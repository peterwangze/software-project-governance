# 生命周期规则

本文件定义 `software-project-governance` workflow 的生命周期规则。

## 阶段列表

1. 立项（Initiation）
2. 规划（Planning）
3. 设计（Design）
4. 开发（Implementation）
5. CI（Verification）
6. 验收（Acceptance）
7. 运营（Operation）
8. 维护（Maintenance）

## 阶段规则

### 立项
- 目标：明确问题、目标、边界和成功标准。
- 输出：目标说明、范围边界、成功标准。
- Gate：G1

### 规划
- 目标：建立任务拆解、主计划入口、里程碑和初始风险。
- 输出：计划跟踪表、风险初版。
- Gate：G2

### 设计
- 目标：形成可执行方案和规则结构。
- 输出：生命周期、Gate、结构设计、关键决策。
- Gate：G3

### 开发
- 目标：产出工作流实际资产，如模板、适配入口、脚本。
- 输出：模板、适配器、脚本、文档资产。
- Gate：G4

### CI
- 目标：校验 workflow 资产是否完整且一致。
- 输出：校验结果、问题清单。
- Gate：G5

### 验收
- 目标：确认 workflow 能否作为 agent plugin/skill 样板使用。
- 输出：验收结论、遗留问题清单。
- Gate：G6

### 运营
- 目标：跟踪实际使用反馈与偏差。
- 输出：使用反馈、问题归档、优化建议。
- Gate：G7

### 维护
- 目标：基于复盘和反馈持续修订 workflow。
- 输出：规则修订、模板更新、兼容性扩展。
- Gate：G8

## 统一要求

- 任意时刻只能有一个主阶段。
- 阶段切换前必须检查对应 Gate。
- 偏差必须同步进入风险或决策记录。
- 已完成事项必须能追溯到证据。
