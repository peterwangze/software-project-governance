# 防护网与 CI/CD — 子工作流

本文件定义"防护网与 CI/CD"阶段的独立子工作流。用户可单独加载本文件执行 CI/CD 配置阶段。

## 阶段目标

建立持续集成和持续部署的自动化防护网。本阶段的核心产出是"稳定的 CI 流水线"——每次代码变更自动触发构建、测试和校验，质量门禁自动生效。

## 进入条件

- 测试阶段已完成（G7 通过）
- 有基础 CI 配置（G4 阶段已建立）

## 活动清单

### 1. CI 流水线强化

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 补全测试步骤 | 单元测试 + 集成测试 + 回归测试 | 自动 |
| 补全静态分析 | lint + type check + 代码规范检查 | 自动 |
| 配置构建缓存 | 加速 CI 执行 | 自动 |
| 验证 CI 稳定性 | 最近 5 次运行成功率 ≥ 80% | 自动 |

### 2. 质量门禁配置

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 定义门禁规则 | 覆盖率阈值、lint 规则、安全扫描 | 自动：基于 profile 阈值，需用户确认 |
| 门禁生效验证 | 门禁检查在 CI 日志中可见 | 自动 |
| 配置失败通知 | 门禁不通过时通知相关人 | 需用户输入：通知渠道 |

### 3. 自动化测试套件

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 核心路径自动化 | 核心功能路径的自动化测试 | 自动 |
| 回归测试套件 | 每次变更自动运行 | 自动 |
| 覆盖率报告 | 自动生成并作为门禁依据 | 自动 |

### 4. 部署流程文档化

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 编写部署文档 | 部署步骤、环境要求、回滚方案 | 自动 |
| 部署自动化 | 部署脚本或 CI 部署步骤 | 自动 |
| 验证部署可执行 | 文档步骤可执行 | 自动 |

## 产出物标准

| 产出物 | 格式要求 | 质量标准 |
|--------|---------|---------|
| CI 配置 | 流水线定义文件 | 稳定运行（成功率 ≥ 80%） |
| 质量门禁 | 门禁规则文件 | 门禁检查在 CI 中可见 |
| 自动化测试套件 | 覆盖核心功能路径 | 与设计文档功能模块对应 |
| 部署文档 | 步骤可执行 | 有具体命令和验证方式 |

## 退出条件

- [ ] CI 流水线稳定运行（最近 5 次成功率 ≥ 80%）
- [ ] 自动化测试覆盖核心路径
- [ ] 质量门禁生效
- [ ] 部署流程已文档化

## Gate 映射

本阶段对应 **G8 — 防护网就绪**。

| Gate 检查项 | 对应活动 | 判定标准 |
|------------|---------|---------|
| CI 流水线是否稳定运行？ | CI 流水线强化 | 最近 5 次运行成功率 ≥ 80% |
| 自动化测试是否覆盖核心路径？ | 自动化测试套件 | 路径列表与设计文档功能模块对应 |
| 质量门禁是否生效？ | 质量门禁配置 | 规则文件存在且 CI 日志可见检查 |
| 部署流程是否文档化？ | 部署流程文档化 | 部署文档存在且步骤可执行 |
