# 运营与反馈 — 子工作流

本文件定义"运营与反馈"阶段的独立子工作流。用户可单独加载本文件执行运营阶段。

## 阶段目标

收集真实运行数据和用户反馈，为持续改进提供事实依据。运营阶段的核心产出是"运营报告"——用数据回答"上线后表现如何、用户怎么用、有什么问题"。

## 进入条件

- 版本已发布（G9 通过）
- 有监控和反馈收集机制

## 活动清单

### 1. 数据收集

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 收集运行指标 | 性能、可用性、错误率、用户活跃度 | 自动 |
| 收集用户反馈 | 用户反馈渠道（工单、评论、调研） | 需用户输入：反馈渠道和关键指标 |
| 建立基线对比 | 上线前 vs 上线后数据对比 | 自动 |

### 2. 问题分类

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 问题识别 | 从数据和反馈中发现问题 | 自动 |
| 严重度分类 | 按影响范围和紧急程度分类 | 自动 |
| 分配处理人 | 每个问题有负责人 | 需用户确认 |

### 3. 运营分析

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 效果评估 | 目标达成情况 | 自动 |
| 用户行为分析 | 用户如何使用、哪些功能最常用 | 需用户判断：分析方向 |
| 优化建议 | 基于数据提出改进方向 | 自动，需用户确认优先级 |

### 4. 运营报告

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 编写运营报告 | 数据 + 分析 + 建议 | 自动 |
| 归档反馈 | 按类别归档 | 自动 |

## 产出物标准

| 产出物 | 格式要求 | 质量标准 |
|--------|---------|---------|
| 运营报告 | 包含运行数据和用户反馈 | 至少 1 周的真实运行数据 |
| 问题清单 | 每条有严重级别和状态 | 分类清晰，有处理人 |
| 优化建议 | 至少 1 条可执行的改进方向 | 有数据支撑 |

## 退出条件

- [ ] 收集到真实运营数据
- [ ] 用户反馈已归档
- [ ] 关键问题已识别和分类
- [ ] 优化方向已明确

## Gate 映射

本阶段对应 **G10 — 运营回收**。

| Gate 检查项 | 对应活动 | 判定标准 |
|------------|---------|---------|
| 是否收集到真实运营数据？ | 数据收集 | 运营报告包含至少 1 周的真实运行数据 |
| 用户反馈是否已归档？ | 数据收集 | 反馈已归档，有分类 |
| 关键问题是否已识别和分类？ | 问题分类 | 问题清单存在，每条有严重级别和状态 |
| 优化方向是否已明确？ | 运营分析 | 优化建议列表非空 |
