# 发布 Checklist

本 skill 用于版本发布前的系统化检查，确保发布内容、回滚方案和验证计划就绪。可独立使用。

## 触发条件

- 版本发布前，需要确认发布就绪状态
- 功能已通过测试（G7 通过），准备进入发布流程
- 用户要求对当前版本做发布前检查

## 输入

| 输入项 | 说明 | 是否必须 |
|--------|------|---------|
| 变更范围 | 本次发布包含的功能和修改 | 必须 |
| 测试报告 | 测试阶段的质量评估结果 | 必须 |
| 发布计划（如有） | 已有的发布时间窗口和步骤 | 推荐 |

## 执行步骤

### 第一步：发布范围确认

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 1 | 版本号已定义 | 遵循语义化版本（major.minor.patch） |
| 2 | 变更范围已列出 | 所有变更项有清单（commit 列表或 issue 列表） |
| 3 | 变更类型已标注 | 每项变更标注：新功能 / 修复 / 优化 / 破坏性变更 |
| 4 | 发布时间窗口已确定 | 有明确的发布时间计划 |

### 第二步：变更日志检查

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 1 | CHANGELOG 已更新 | 覆盖本次所有变更（commit 列表与 changelog 对照一致） |
| 2 | 破坏性变更已高亮 | 不兼容变更有明确说明和迁移指引 |
| 3 | 依赖变更已记录 | 新增或升级的依赖有版本号和变更原因 |
| 4 | 已知问题已列出 | 已知但未修复的问题有说明和 workaround |

### 第三步：回滚方案验证

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 1 | 回滚方案已编写 | 有具体的回滚步骤（不是"回滚到上一版本"这样的泛化描述） |
| 2 | 回滚方案已验证 | 在测试环境执行过回滚操作，确认可成功 |
| 3 | 数据兼容性 | 回滚后数据是否兼容（数据迁移场景必检） |
| 4 | 回滚影响范围 | 回滚操作的影响已评估，不会造成额外损失 |

**参考 Amazon 可逆性分类**：
- **可逆发布**（灰度/回滚容易）：快速推进，回滚方案简化
- **不可逆发布**（数据迁移/Schema 变更）：深度论证回滚方案，必须有验证记录

### 第四步：发布后验证计划

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 1 | 核心功能验证清单 | 列出发布后需要验证的核心功能项 |
| 2 | 监控指标基线 | 发布前的关键指标已记录（用于对比） |
| 3 | 告警规则就绪 | 发布相关的监控告警已配置 |
| 4 | 验证责任人 | 每项验证有明确的负责人 |

### 第五步：发布决策

| 决策 | 条件 | 后续动作 |
|------|------|---------|
| 可以发布 | 全部检查项通过 | 执行发布 |
| 有条件发布 | 非关键项有遗留 | 记录遗留项，发布后 48 小时内关闭 |
| 暂缓发布 | 关键检查项未通过 | 修复后重新检查 |

## 输出

| 产出物 | 格式 |
|--------|------|
| 发布 checklist 完成状态 | 每项检查的通过/未通过状态 |
| 变更日志 | 本次发布的完整变更记录 |
| 回滚方案 | 具体步骤，已在测试环境验证 |
| 发布后验证清单 | 核心功能验证项 + 责任人 |
| 发布决策 | 可以发布 / 有条件发布 / 暂缓发布 |

## 质量标准

- [ ] 发布范围明确，有版本号和变更清单
- [ ] 变更日志覆盖所有变更，破坏性变更已高亮
- [ ] 回滚方案有具体步骤且已验证
- [ ] 发布后验证清单存在且包含核心功能验证项
- [ ] 发布决策有明确理由

## 独立使用说明

本 skill 可脱离完整生命周期独立使用。

### 如果项目已初始化 governance（`.governance/` 存在）
1. 先执行目标锚定检查（读 plan-tracker → 确认成功标准 → 偏离检查）
2. 不需要先通过任何 Gate
3. 可用于任何版本发布前的检查（不限于正式发布阶段）
4. 热修复/紧急发布也可使用本 checklist，但可跳过非必要检查项并记录跳过原因
5. 如果后续接入完整工作流，本 skill 的产出自动成为 G9（发布就绪）的证据

### 如果项目未初始化 governance（`.governance/` 不存在）
1. 跳过所有治理前置检查
2. 直接执行本 skill 的核心流程（发布前 checklist + 回滚方案验证 + 发布决策）
3. 执行完毕后提醒用户："当前项目未初始化 governance。如需自动证据记录、Gate 检查、风险跟踪，运行 `/governance-init`。"
