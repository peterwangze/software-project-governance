# 环境搭建与基础设施 — 子工作流

本文件定义"环境搭建与基础设施"阶段的独立子工作流。用户可单独加载本文件执行环境搭建阶段。

## 阶段目标

建立可复现的开发环境和基础设施。环境搭建阶段的核心产出是"环境可用"——新加入的成员可以在合理时间内完成环境配置并开始开发。

## 独立使用时的目标锚定

> 当本子工作流被独立加载（非通过 `main-workflow.md` 总入口加载）时，**MUST** 在执行阶段活动前完成：

1. **目标确认**：读取 `.governance/plan-tracker.md`，确认项目立项时定义的量化成功标准
2. **偏离检查**：对照成功标准，确认环境搭建范围与项目总目标一致。如发现偏离，**MUST** 先记录到 `.governance/decision-log.md` 再继续
3. **质量底线**：环境配置必须可复现、可文档化（参考 profile 质量要求）

## 进入条件

- 技术选型已完成（G3 通过）
- 有明确的技术栈决策

## 活动清单

### 1. 仓库初始化

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 创建仓库结构 | 按约定建立目录结构 | 自动 |
| 初始化版本控制 | git init + .gitignore + 初始提交 | 自动 |
| 配置分支策略 | main/develop/feature 或 trunk-based | 需用户确认：分支策略 |

### 2. 开发环境配置

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 编写环境配置文档 | 依赖安装、环境变量、本地配置 | 自动 |
| 提供环境复现手段 | Dockerfile / 一键脚本 / 环境配置文件 | 自动 |
| 配置编辑器/IDE 规范 | 格式化、lint、推荐插件 | 自动 |

### 3. 基础 CI 配置

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 配置 CI 流水线 | 自动构建 + 基础检查 | 自动 |
| 配置 lint/format | 代码风格自动检查 | 自动 |
| 验证 CI 可运行 | 推送后确认 CI 通过 | 自动 |

### 4. 协作规范建立

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 编写贡献指南 | 提交规范、PR 流程、Review 规则 | 自动 |
| 配置提交检查 | commit message 格式、pre-commit hooks | 自动 |
| 用户确认环境可用 | 新成员能否成功配置环境 | 需用户确认 |

## 产出物标准

| 产出物 | 格式要求 | 质量标准 |
|--------|---------|---------|
| 仓库结构 | 目录结构与约定一致 | 验证脚本通过 |
| 环境配置文档 | 包含完整安装步骤 | 新机器可复现 |
| CI 配置 | 基础构建 + 检查可运行 | 推送后 CI 通过 |
| 协作规范 | 贡献指南存在 | 至少覆盖提交和 PR 规范 |

## 退出条件

- [ ] 开发环境可复现（有 Dockerfile/环境配置文档/一键脚本）
- [ ] 仓库结构与约定一致
- [ ] 基础 CI 可运行
- [ ] 协作规范已建立

## Gate 映射

本阶段对应 **G4 — 环境就绪**。

| Gate 检查项 | 对应活动 | 判定标准 |
|------------|---------|---------|
| 开发环境是否可复现？ | 开发环境配置 | 有 Dockerfile/环境配置文档/一键脚本 |
| 仓库结构是否符合约定？ | 仓库初始化 | 验证脚本通过 |
| 基础 CI 是否可运行？ | 基础 CI 配置 | CI 配置文件存在且运行成功 |
| 协作规范是否已建立？ | 协作规范建立 | 贡献指南/编码规范/提交规范至少 1 份 |
