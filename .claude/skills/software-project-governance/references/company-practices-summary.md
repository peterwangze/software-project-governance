# 企业实践经验 — 摘要

本文件是从 `workflows/software-project-governance/research/company-practices.md` 提取的摘要，供自包含插件快速参考。完整版请查阅 workflows/ 本体层中的原文。

## 覆盖企业

| 企业 | 核心实践 | 主要映射阶段 |
|------|---------|------------|
| **Google** | Design Doc、Code Review、OKR、Blameless Postmortem | 立项、架构设计、开发、维护 |
| **Amazon** | Working Backwards、6-Pager、PR/FAQ、Bar Raiser | 立项、技术选型、架构设计、开发 |
| **华为** | IPD 门控、蓝军机制、自我批判、让听得见炮声的人呼唤炮火 | 立项、架构设计、测试、维护 |
| **字节跳动** | OKR 全员公开、A/B 测试、数据驱动、Always Day 1 | 立项、开发（AB实验）、运营、维护 |

## 使用方式

- **立项阶段**：参考 Google OKR / Amazon Working Backwards 定义目标
- **架构设计阶段**：参考 Google Design Doc / Amazon 6-Pager 撰写设计文档
- **开发阶段**：参考 Google Code Review / 华为蓝军机制进行代码审查
- **测试阶段**：参考华为 IPD 门控 / 字节 A/B 测试文化设计测试策略
- **维护阶段**：参考 Google Blameless Postmortem / 华为自我批判进行复盘

完整企业实践（含每条实践的适用条件、具体规则和子工作流映射）见 `workflows/software-project-governance/research/company-practices.md`。
