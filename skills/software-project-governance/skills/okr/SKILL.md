# OKR 模板 — Google OKR + ByteDance 公开透明

本模板用于立项阶段定义可衡量的项目目标。基于 Google OKR 和 ByteDance OKR 全员公开实践。

## 使用时机

立项阶段，与 PR/FAQ 配合使用。PR/FAQ 定义用户价值，OKR 定义可衡量的成功标准。

## Objective（目标）

**用一句话说清方向。"去哪里"而非"怎么去"。**

```
O: [定性的方向描述]

为什么这个目标重要：
[1-2 句话说明与上级目标/公司战略的对齐关系]
```

**质量检查**：
- [ ] 目标是一个方向，不是一个任务
- [ ] 读给不了解项目的人听，他们能理解"我们要达成什么"
- [ ] 没有使用技术实现语言（如"搭建 XX 系统"）——那是 KR 的事

## Key Results（关键结果）

**3-5 个可衡量的结果。每个 KR 达成 = 目标达成。**

| # | Key Result | 基线值 | 目标值 | 衡量方式 | 截止日期 |
|---|-----------|--------|--------|---------|---------|
| KR1 | [可衡量的结果描述] | [当前是多少] | [要达到多少] | [怎么衡量] | [日期] |
| KR2 | | | | | |
| KR3 | | | | | |

**KR 质量检查**：
- [ ] 每个 KR 有数字（百分比/绝对值/二值判定）
- [ ] 有基线数据——"基线值"列不能为空（新项目可写"0"或"不存在"）
- [ ] KR 达成 = O 达成（不像"提升用户满意度"→ KR "写 3 篇博客"——博客不能证明满意度）
- [ ] 衡量方式可自动检查或可独立验证——不接受"团队感觉"

## 向上对齐

```
本项目目标对齐到：[上级目标/公司 OKR/独立目标声明]
上级目标的 KR：[相关的上级 KR]
本项目对其的贡献：[本项目的哪个 KR 支撑了上级的哪个 KR]
```

**如果是独立目标**：声明 `本项目为独立目标，无上级 OKR 对齐关系。` 独立目标是合法的——但要显式声明。

## 可见性

- [ ] OKR 已写入项目的事实源（`.governance/plan-tracker.md` 或独立文件）——项目所有成员可访问
- [ ] OKR 的进度在每次复盘时更新
- [ ] 目标调整时 MUST 记录到 decision-log

## 企业实践来源

| 实践 | 来源 | 在本模板中的体现 |
|------|------|----------------|
| OKR 格式（O + KR × 3-5） | Google | O 定性 + KR 定量结构 |
| 基线数据 | ByteDance | 每个 KR 必须有基线值，不接受"没有数据" |
| 全员公开 | ByteDance | 可见性检查——OKR 所有人可访问 |
| 向上追溯 | ByteDance | 向上对齐节——本项目的 O 对应哪个上级 O |
