# Stage Gate Rules

本文件定义 `software-project-governance` workflow 的阶段门禁规则。

## G1 - 立项完成

- 阶段转换：立项 -> 规划
- 必审材料：目标、范围边界、成功标准
- 检查项：问题是否清晰；目标是否可衡量；是否明确启动价值
- 通过标准：立项基础信息完整，无关键歧义

## G2 - 规划完成

- 阶段转换：规划 -> 设计
- 必审材料：计划跟踪表、任务拆解、风险初版
- 检查项：是否建立统一计划入口；是否识别依赖和里程碑；是否识别初始风险
- 通过标准：计划具备执行性，责任人和任务可跟踪

## G3 - 设计完成

- 阶段转换：设计 -> 开发
- 必审材料：生命周期定义、Gate 规则、关键决策
- 检查项：结构是否可执行；规则是否完整；关键权衡是否留痕
- 通过标准：方案可执行，关键不确定性已显式管理

## G4 - 开发完成

- 阶段转换：开发 -> CI
- 必审材料：模板、适配入口、脚本、偏差记录
- 检查项：资产是否齐备；是否与规则对齐；是否可被检查
- 通过标准：workflow 资产完整，可进入验证环节

## G5 - CI 完成

- 阶段转换：CI -> 验收
- 必审材料：校验结果、问题清单、关键证据
- 检查项：校验脚本是否通过；是否存在阻塞性缺陷；关键证据是否补齐
- 通过标准：质量达到进入验收的最低标准

## G6 - 验收完成

- 阶段转换：验收 -> 运营
- 必审材料：验收结论、遗留问题清单、适配说明
- 检查项：是否满足首版 plugin/skill 目标；是否可供 Claude/Codex 使用
- 通过标准：workflow 可作为首版插件样板被消费

## G7 - 运营回收

- 阶段转换：运营 -> 维护
- 必审材料：使用反馈、问题记录、改进建议
- 检查项：是否收集到真实反馈；是否识别兼容问题；是否形成维护动作
- 通过标准：反馈已沉淀，可进入维护

## G8 - 维护闭环

- 阶段转换：维护 -> 下一轮立项或规划
- 必审材料：复盘结论、模板修订、兼容扩展记录
- 检查项：是否完成复盘；是否回灌规则；是否更新扩展计划
- 通过标准：形成版本化演进闭环

## Gate 执行原则

- 未通过 Gate，不得声称进入下一阶段。
- Gate 结论必须能回溯到证据和事实源。
- 任何 agent 适配都不得绕过 Gate 规则。
