# 测试与质量保障 — 子工作流

本文件定义"测试与质量保障"阶段的独立子工作流。用户可单独加载本文件执行测试阶段。

## 阶段目标

验证系统满足功能和非功能需求。测试阶段的核心产出是"测试报告"——用数据说明系统质量是否达到发布标准。

## 进入条件

- 开发已完成（G6 通过或条件通过）
- 有测试计划或测试策略

## 活动清单

### 1. 测试计划确认

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 确认测试范围 | 哪些功能需要测试、哪些可以跳过 | 需用户确认：测试优先级 |
| 确认测试类型 | 功能测试、性能测试、安全测试、兼容性测试 | 自动：基于非功能需求推荐，需用户确认 |
| 确认通过标准 | 缺陷密度、覆盖率、性能阈值 | 需用户确认 |

### 2. 功能测试

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 测试用例设计 | 基于需求和设计文档编写测试用例 | 自动 |
| 功能测试执行 | 逐用例执行并记录结果 | 自动 |
| 缺陷记录 | 每个缺陷有严重级别、复现步骤、预期结果 | 自动 |

### 3. 非功能测试

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 性能测试 | 响应时间、吞吐量、资源占用 | 自动：执行性能测试套件 |
| 安全测试 | OWASP Top 10 关键项 | 自动：运行安全扫描 |
| 兼容性测试 | 浏览器/设备/平台兼容性 | 需用户输入：目标平台列表 |

### 4. 测试报告

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 汇总测试结果 | 通过率、缺陷统计、覆盖率 | 自动 |
| 评估质量状态 | 是否达到发布标准 | 自动：与 profile 阈值对比 |
| 输出测试报告 | 包含量化结果和结论 | 自动 |

## 产出物标准

| 产出物 | 格式要求 | 质量标准 |
|--------|---------|---------|
| 测试计划 | 覆盖功能和非功能测试 | 测试范围和通过标准明确 |
| 测试用例 | 每个用例有前置条件、步骤、预期结果 | 覆盖核心功能路径 |
| 缺陷清单 | 每个缺陷有严重级别和状态 | 关键/阻塞级别缺陷数 = 0 |
| 测试报告 | 包含量化结果 | 通过率、覆盖率、性能数据齐全 |

## 退出条件

- [ ] 关键缺陷已关闭（关键/阻塞级别 = 0）
- [ ] 回归测试通过（通过率 = 100%）
- [ ] 性能指标达标
- [ ] 安全测试覆盖关键风险

## Gate 映射

本阶段对应 **G7 — 测试通过**。

| Gate 检查项 | 对应活动 | 判定标准 |
|------------|---------|---------|
| 关键缺陷是否已关闭？ | 缺陷记录 | 关键/阻塞级别缺陷数 = 0 |
| 回归测试是否通过？ | 功能测试 | 回归测试通过率 = 100% |
| 性能指标是否达标？ | 性能测试 | 性能测试结果在目标范围内 |
| 安全测试是否覆盖关键风险？ | 安全测试 | OWASP Top 10 关键项已覆盖，无高危漏洞 |
