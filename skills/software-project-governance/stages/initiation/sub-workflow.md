# 立项与目标定义 — 子工作流

本文件定义"立项与目标定义"阶段的独立子工作流。用户可单独加载本文件执行立项阶段，不强制加载全流程。

## 阶段目标

明确问题、目标、边界、成功标准和关键约束。立项阶段的核心产出是一份所有干系人对齐的"项目章程"——回答"做什么、为什么做、不做什么、怎么算成功"。

## 进入条件

- 有一个待解决的问题或机会
- 至少一个发起人愿意对项目结果负责

## 活动清单

### 1. 问题陈述

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 描述当前痛点 | 用 1~3 句话说明现状有什么问题 | 需用户输入：痛点是什么 |
| 量化问题影响 | 说明问题不解决的代价（时间、成本、质量） | 需用户输入：影响有多大 |
| 确认问题真实存在 | 验证问题不是假设，有事实支撑 | 需用户确认 |

### 2. 目标定义

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 定义项目目标 | 用一句话描述项目要达成的结果 | 需用户输入：目标是什么 |
| 量化成功标准 | 为目标定义 2~5 个可衡量的指标 | 需用户输入：怎么算成功 |
| 确认目标可衡量 | 检查每个指标是否可量化或可判定 | 自动：检查指标是否包含量词或判定标准 |

### 3. 范围边界

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 列出范围内事项 | 明确本项目要做什么 | 需用户输入 |
| 列出范围外事项 | 明确本项目**不做什么** | 需用户输入 |
| 检查范围清晰度 | 确认范围内外没有歧义 | 自动：检查是否有明确的"不做什么"列表 |

### 4. 干系人识别

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 列出关键干系人 | 识别对项目结果有影响或受影响的角色 | 需用户输入 |
| 确认干系人角色 | 每个干系人的职责和期望 | 需用户确认 |

### 5. 初始风险评估

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 识别初始风险 | 列出已知的阻塞和不确定性 | 需用户输入 + 自动补充常见风险 |
| 评估风险严重度 | 对每个风险评估概率和影响 | 自动：按标准矩阵判定 |

## 产出物标准

| 产出物 | 格式要求 | 质量标准 |
|--------|---------|---------|
| 项目章程 | 文档，包含目标、范围、干系人、成功标准 | 目标可衡量、范围有明确的"不做什么" |
| 初始风险清单 | 至少 3 条风险，每条有严重度评估 | 每条风险有触发条件和缓解方向 |
| 干系人列表 | 包含角色、职责、期望 | 所有关键角色已覆盖 |

## 退出条件

- [ ] 项目目标已定义且可衡量（至少 1 个量化指标）
- [ ] 范围边界已明确（有"不做什么"列表）
- [ ] 关键干系人已识别且角色已确认
- [ ] 初始风险已记录（至少 3 条）

## Gate 映射

本阶段对应 **G1 — 立项完成**。

| Gate 检查项 | 对应活动 | 判定标准 |
|------------|---------|---------|
| 项目目标是否可衡量？ | 目标定义 | 成功标准至少包含 1 个量化指标 |
| 范围边界是否清晰？ | 范围边界 | 有明确的"不做什么"列表 |
| 关键干系人是否已识别？ | 干系人识别 | 干系人列表非空，每条有角色和职责 |
| 是否有明确的"不做什么"清单？ | 范围边界 | 范围外事项列表非空 |
