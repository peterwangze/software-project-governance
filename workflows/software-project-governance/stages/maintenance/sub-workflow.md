# 维护与演进 — 子工作流

本文件定义"维护与演进"阶段的独立子工作流。用户可单独加载本文件执行维护阶段。

## 阶段目标

通过复盘和经验回灌持续改进工作流和项目。维护阶段的核心产出是"复盘结论和改进计划"——回答"学到了什么、哪里做得不好、下次怎么做得更好"。

## 进入条件

- 运营阶段已完成（G10 通过）
- 有运营报告和反馈数据

## 活动清单

### 1. 复盘执行

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 目标回顾 | 逐项评估目标完成度 | 自动 |
| 结果评估 | 做得好/做得不好各至少 3 条 | 自动 + 需用户判断：补充视角 |
| 根因分析 | 对关键问题做 5-Why 追溯 | 自动 |

### 2. 经验回灌

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 提炼可复用经验 | 按维度分类：流程/技术/协作/管理 | 自动 |
| 映射到规则和模板 | 经验能否转化为检查项或模板字段 | 自动 |
| 更新工作流资产 | 按映射结果更新子工作流、Gate 或 skill | 自动 |

### 3. 下轮规划

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 确定下轮方向 | 下一轮迭代的目标和优先级 | 需用户确认 |
| 制定改进计划 | 不超过 5 条改进项，每条有责任人和截止日期 | 自动，需用户确认 |
| 更新计划跟踪表 | 改进项进入下一轮跟踪 | 自动 |

### 4. 版本化更新

| 活动 | 描述 | 交互边界 |
|------|------|---------|
| 更新版本记录 | 版本号、日期、变更说明 | 自动 |
| 更新样例总览 | 统计数据重新计算 | 自动 |

## 产出物标准

| 产出物 | 格式要求 | 质量标准 |
|--------|---------|---------|
| 复盘文档 | 目标回顾 + 结果评估 + 根因分析 + 经验沉淀 + 改进计划 | 每部分有实质内容 |
| 改进计划 | 不超过 5 条，每条有责任人和截止日期 | 可执行 |
| 资产更新记录 | 哪些文件被更新、更新了什么 | 与 git 提交记录对应 |

## 退出条件

- [ ] 复盘已完成
- [ ] 经验已回灌到规则和模板
- [ ] 下轮方向已明确
- [ ] 版本化记录已更新

## Gate 映射

本阶段对应 **G11 — 维护闭环**。

| Gate 检查项 | 对应活动 | 判定标准 |
|------------|---------|---------|
| 是否完成复盘？ | 复盘执行 | 复盘文档存在，包含目标回顾、结果评估、原因分析、经验沉淀 |
| 经验是否回灌到规则和模板？ | 经验回灌 | 规则/模板有对应的更新提交记录 |
| 下轮方向是否已明确？ | 下轮规划 | 下一轮方向在计划跟踪表中有 ≥1 条 P0 任务 |
| 版本化记录是否已更新？ | 版本化更新 | manifest 或样例总览中的版本/日期字段已更新 |
